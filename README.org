#+TITLE: sched

Scheduler for ML jobs across my machines. This works simply by having a network
of machines sharing /tasks/ and /results/ together via any file sharing protocol. A
task is a structured file that specifies, for now, a problem for LLM to solve
along with some requirements on the model. A machine that has the capability and
time to solve this solves them and writes out corresponding result file for
syncing.

For my use case, I am starting with creating a shared syncthing directory for
tasks and results. The worker is a simple binary running on my machines that can
run decent sized LLMs. This repository houses the worker code.

To run the worker, first ensure you have ~ollama~ installed. Then run:

#+begin_src shell
  sched ./path/to/tasks/dir
#+end_src

Currently it defaults to a hardcoded model ~deepseek-r1:1.5b~ but this will
change. Mostly we will allow providing model name via an environment variable or
as an specification in the task themselves.
